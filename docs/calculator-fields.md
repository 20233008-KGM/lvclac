# 계산기 입력·증거금 설계 정리

> 지금까지 채팅에서 나온 논의(통화 단위, 국내/해외, 약정금액, 1포인트당 손익, 필드 배치 등)를 **현재 코드 기준**으로 한곳에 정리한 문서입니다.

## 1. 기본 원칙

### 통화(원·달러)를 넣지 않는 이유

- 계산기는 **브로커 앱에 보이는 숫자를 그대로** 넣는 도구입니다.
- 계좌 평가금액·약정금액·증거금이 전부 **같은 암묵적 단위**(원이든 달러든)이면, 내부 연산은 비율·대수만 쓰므로 **환율·통화 표시가 필요 없습니다**.
- UI도 `₩`/`$` 없이 숫자만 표시합니다.

### 브로커마다 규정이 다름

- 청산 시점, 반올림, 수수료, 유지증거금 산식은 증권사·거래소마다 다를 수 있습니다.
- 본 계산기는 **참고용 추정**이며, HTS·MTS와 100% 일치를 보장하지 않습니다.

## 2. 용어 정리 (헷갈리기 쉬운 것)

| 용어 | 뜻 | 비고 |
|------|-----|------|
| **계좌 평가금액** | 예탁금 + 미결제손익 | HTS 「계좌 평가금액」과 동일 |
| **약정금액 (1계약)** | 종목 스펙상 1계약 명목·약정 금액 | 영문 UI: Notional per contract. **1계약 명목가치와 같은 입력값** |
| **약정가치** | 포지션 전체 규모 | `약정금액 × 계약승수 × 보유 계약수` (결과 패널) |
| **계약 승수** | 약정금액 산출용 배율 | 비우면 **1**. 거래소 스펙의 「1포인트당 N원」과 UI 필드 이름이 1:1은 아님 (아래 §4) |
| **1포인트당 손익 (1계약)** | 시세가 1포인트 움직일 때 **1계약** 손익 | 미니 코스피 예: **5만원/포인트 (고정)**. **현재 UI 입력란 없음** — 약정금액·현재가로 자동 산출 |
| **보유 계약수** | 현재 포지션 계약 수 | 1포인트당 손익을 정의하는 값이 **아님** |
| **포지션 전체 1포인트 손익** | `1포인트당 손익(1계약) × 보유 계약수` | 청산가 계산에 실제로 쓰는 민감도 |

### 자주 틀리기 쉬운 구분

```
❌ 1포인트당 손익 = 보유 계약수 × 계약승수 × 1   (틀림 — 계약수는 “1포인트당” 정의와 무관)

✅ 1포인트당 손익 (1계약) = 거래소 계약단위로 고정  (예: 미니 5만원/포인트)

✅ 포지션 전체 1포인트 손익 = 1포인트당 손익(1계약) × 보유 계약수
```

## 5. 계산 공식 (코드와 동일)

코드 위치: `src/calc/pointValue.ts`, `src/calc/margins.ts`, `src/calc/leverage.ts`

### 5.1 1포인트당 손익 (1계약) — 내부 자동 계산

```
1포인트당 손익 = 약정금액 × 계약승수 ÷ 현재가
```

(계약승수 미입력 시 1)

**한국 실무:** 거래소 스펙상 1포인트당 손익은 **고정**입니다 (미니 5만원/포인트 등).  
HTS에서 **약정금액과 현재가를 같이** 가져오면, 위 나눗셈 결과 ≈ 그 고정값입니다.

```
예: 미니, 지수 350, 1포인트당 5만원 고정
    HTS 약정금액 ≈ 17,500,000  (≈ 350 × 50,000)
    17,500,000 ÷ 350 = 50,000  ✓
```

**주의:** 약정금액만 옛날 값으로 두고 현재가만 바꾸면 나눗셈 결과가 어긋납니다.  
→ **HTS에 보이는 약정금액·현재가를 한 세트로** 넣는 것이 맞습니다.

### 5.2 약정가치 (포지션 명목)

두 경로는 수학적으로 같습니다.

```
약정가치 = 보유 계약수 × 현재가 × 1포인트당 손익
         = 보유 계약수 × 약정금액 × 계약승수
```

### 5.3 증거금

**직접 입력**이 있으면 비율보다 우선. 주문 시뮬레이션 시 계약 수에 비례해 확장.

| 종류 | 비율 사용 시 | 직접 입력 시 |
|------|-------------|-------------|
| 유지증거금 | 약정가치 × 유지증거금률 | 앱 표시 금액 |
| 위탁증거금 | 약정가치 × 개시증거금률 | 앱 표시 금액 |

### 5.4 청산가

```
여유 = 계좌 평가금액 − 유지증거금
포인션 1포인트 손익 = 1포인트당 손익(1계약) × 보유 계약수
가격 변동폭 = 여유 ÷ 포인션 1포인트 손익

롱 청산가 ≈ 현재가 − 가격 변동폭
숏 청산가 ≈ 현재가 + 가격 변동폭
```

## 6. 한국 vs 해외 — 무엇이 다른가

**차이는 통화가 아니라 브로커 화면에 뭐가 적혀 있느냐**입니다.

| | 한국 HTS·MTS | 해외 브로커 (예: CME) |
|--|-------------|----------------------|
| 계좌·계약수·현재가 | ✓ | ✓ |
| 약정금액 / 1계약 명목 | ✓ 「약정금액」 | ✓ Notional, contract value 등 |
| 증거금 | 비율 + 금액 둘 다 흔함 | **고정 $금액**만 보는 경우 많음 → **직접 입력** |
| 1포인트당 손익 | 거래소 고정값; UI 필드 없이 약정금액÷현재가로 충분 | 스펙에 $50/point 등으로 명시 |

### 「1포인트당 손익」 입력란을 뺀 이유

- 한국: **약정금액 + 현재가 (+ 계약승수)** 만으로 자동 계산됨.
- 해외: 대부분 **명목가치·증거금·현재가**를 볼 수 있고, $/point만 알 때는 `약정금액 ≈ (1포인트당 손익) × 현재가` 한 번 넣으면 동일.
- 필드를 늘리면 오히려 「뭘 넣어야 하지?」 혼란이 커짐 → **제거하고 단순화**.

### 나중에 다시 넣을지

해외 트래픽이 늘고 「약정금액 칸에 뭘 넣는지」 문의가 많아지면:

- **1포인트당 손익 (선택)** 복구
- **약정금액과 둘 중 하나만** 채우면 되게 (`약정금액 XOR 1포인트당 손익`)

현재는 단순화 우선.

## 8. 채팅에서 나왔던 오해 · 정정

| 말했던 것 | 정정 |
|-----------|------|
| 「1포인트당 손익이 현재가 따라 변한다」 | **1계약 기준 1포인트당 손익은 거래소 스펙으로 고정.** 변하는 것처럼 보인 건, 약정금액을 고정해 두고 현재가만 바꿔서 나눗셈한 **잘못된 사용**일 때뿐. HTS 값을 같이 넣으면 고정값과 일치. |
| 「해외 유저는 조회가 안 돼서 많이 불편」 | **과장됨.** 대부분 equity·계약수·가격·증거금(금액 또는 %) 조회 가능. 이름만 다름. |
| 「계약수 × 계약승수 = 1포인트당 손익」 | **아님.** 계약수는 포지션 크기. 1포인트당 손익(1계약)은 계약단위(거래소 스펙)로 고정. |
| 「1계약 명목가치 ≠ 약정금액」 | **이 계산기에서는 같음.** `contractAmount` 필드 하나. |
