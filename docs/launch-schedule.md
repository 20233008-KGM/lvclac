# LiqGuard 정식 런칭 일정

기준일: 2026-07-15 수요일

목표: **2026-08-07 금요일 정식 공개 + 실결제 시작**

공개 원칙: 무료 베타나 사전 공개를 기본 일정에 두지 않는다. 법인·결제·운영 준비가 모두 끝난 뒤 한 번에 공개한다.

## 일정 판단

- 7/24는 개발만 보면 접근 가능하지만, 법인등기 → 사업자등록 → 정산계좌 → Paddle 사업자 확인의 외부 대기시간을 안정적으로 담지 못한다.
- 8/7은 외부 처리를 가장 빠르게 이어 붙이면서도 실결제 1회와 환불/해지 검증까지 포함할 수 있는 **타이트한 목표일**이다.
- 법인카드 신청과 배송은 병렬 처리한다. 법인카드 실물 수령은 서비스 판매에 필요하지 않으므로 런칭 게이트가 아니다.
- AdSense 승인은 런칭 게이트가 아니다. 8/7 공개 후 신청·승인을 이어가며 승인 전에는 placeholder를 유지한다.

## 핵심 경로

```text
7/16 전자서명·등기 접수
  → 7/17~7/23 등기소 심사
  → 등기 완료 당일 사업자등록 신청
  → 늦어도 7/27 사업자등록 완료
  → 7/28 정산계좌·법인카드 신청 + Paddle 사업자 정보 제출
  → 7/29~8/03 Paddle 확인·live 설정
  → 8/04~8/06 실결제·환불·최종 QA
  → 8/07 한 번에 공개
```

외부기관 답변을 기다리는 동안에는 결제 sandbox 검증, 운영 문서, QA를 진행한다. 대기일을 휴식일이나 빈 날로 계산하지 않는다.

## 사람이 먼저 해야 AI가 이어갈 수 있는 것

아래 항목은 기다리지 말고 해당 날짜의 첫 작업으로 처리한다.

1. **7/16:** 누나와 일괄전자서명 후 등기 신청서를 실제 제출한다.
2. **7/17:** Paddle 계정·sandbox 상품·도메인 심사를 시작한다.
3. **등기 완료 당일:** 등기사항증명서를 발급하고 사업자등록을 즉시 신청한다.
4. **사업자등록 완료 다음 영업일:** 법인 정산계좌와 법인카드를 신청한다.
5. **Paddle 요청 메일 수신 당일:** 추가 서류를 즉시 제출한다.
6. **7/30·8/5:** 본인이 실제 결제·해지/환불 확인을 수행한다.

8/7 전에는 아래 일만 한다.

- 법인·사업자·계좌·Paddle live 전환
- 결제, Pro 권한, 로그인, 저장, 운영 문서의 런칭 차단 문제

- 공유 기능, 신규 통계, 추가 Pro 기능
- 관리자 대시보드 확장
- 롤오버 이메일·푸시 알림

- 런칭을 막지 않는 UI 미세 개선과 리팩터링

## 일정 미끄러짐 규칙

- 외부기관의 예상 완료일은 약속이 아니라 관리 기준이다. 요청이 오면 같은 날 보완한다.
- **8/3까지 Paddle live 승인이 완료되지 않거나 8/5 실결제가 실패하면 8/7 공개 공지를 하지 않는다.** 이미 무료 공개한 상태를 만들지 않고 비공개를 유지한다.
- 이 경우 새 공개일은 승인 완료 뒤 첫 번째 금요일로 다시 확정한다. 1차 비상 후보는 **8/14 금요일**이다.
- 기능 욕심 때문에 날짜를 미루지 않는다. 런칭 게이트에 없는 일은 backlog로 보낸다.

## 매일 마감 체크

1. 사람 작업의 외부 접수번호·상태·추가 요청을 기록한다.
2. AI 작업은 `npm run build`와 관련 테스트를 통과시킨다.
3. GitHub push와 Notion Tasks/Work Log를 최신화한다.
4. 다음 날의 첫 사람 작업과 첫 AI 작업을 각각 한 문장으로 남긴다.

## 공식 처리기간 참고

- 법인등기: 현재 startbiz 안내 기준 접수 후 약 2~5일로 관리한다.
- 법인설립신고·사업자등록: 정부24 안내 처리기간 총 2일. 보정기간은 제외된다.
- Paddle 수동 심사 예상: 도메인 5~7영업일, 사업자 확인 2~4영업일, 신원 확인 1~3영업일. 자동 승인되면 단축되지만 일정은 수동 심사 가능성을 기준으로 관리한다.

이 일정은 법률·세무 자문이 아니다. 사업 목적, 통신판매업 신고, 대표자 보수·4대보험은 세무사 또는 관할기관에 최종 확인한다.
