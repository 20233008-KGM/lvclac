기준일: 2026-07-04 토요일

작업 기준: 주말/평일 구분 없이 매일 작업한다. 요일은 달력에서 찾기 쉽게 붙인 표식일 뿐, 쉬는 날을 의미하지 않는다.

## MVP 범위

이번 런칭에 반드시 포함한다.

| 영역 | 포함 범위 |
| --- | --- |
| 로그인 | 이메일 로그인, 구글 로그인, 로그아웃, 닉네임 |
| DB | `profiles`, `number_sets`, `subscriptions` |
| 유료 기능 | 광고 제거, 숫자세트 저장, 클라우드 동기화 |
| 결제 | 결제 시작, 웹훅 처리, Pro 권한 반영 |
| 운영 | 개인정보처리방침, 이용약관, 환불/해지 안내, 문의 채널 |
| 광고 | AdSense 신청 준비, 승인 전 placeholder 유지 |

이번 런칭에서는 미룬다.

| 미루는 항목 |
| --- |
| 복잡한 요금제 여러 개 |
| lifetime 플랜 |
| 관리자 대시보드 풀버전 |
| 고급 통계 |
| 완전 자동 세무/회계 처리 |

## 리스크 메모

- 결제사는 개발 전에 확정해야 한다. 한국 사업자/한국 정산이면 국내 PG 또는 PortOne 계열 검토가 필요하고, 해외 정산 구조라면 Stripe가 더 자연스러울 수 있다.
- AdSense는 신청 후 승인 대기가 발생한다. 런칭의 필수 조건으로 묶지 말고, 승인 전에는 placeholder를 유지한다.
- 법인설립은 개발을 막지 않는다. 정산, 세금, 환불 정책은 세무사/법무사 확인을 병렬로 진행한다.
