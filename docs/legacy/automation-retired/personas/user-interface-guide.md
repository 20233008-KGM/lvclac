# 사용자용 페르소나 인터페이스 안내

> [!IMPORTANT]
> 퇴역 문서 보존본입니다. 이 파일은 과거 repo 내부 MD 페르소나 시스템의 보존본입니다. 현재 작업 역할과 장기 메모리는 AutoCorp Studio 세션 페르소나/메모리를 기준으로 합니다. 이 문서를 현재 호출 규칙, 작업 지시, 장기 메모리 저장소로 사용하지 않습니다.

이 문서는 퇴역된 회사 부서 페르소나 시스템을 사용자가 어떻게 부르고 활용했는지 설명하던 보존본이다.

## 가장 쉬운 사용법

대부분의 업무는 `총괄관리자`에게 말하면 된다. 총괄관리자는 요청을 정리하고 필요한 부서 관점을 붙여서 하나의 답으로 돌려준다.

예시:

```text
총괄관리자, 회원가입 플로우를 실제 서비스 기준으로 다시 점검하고 고쳐줘.
```

```text
총괄관리자, Pro 결제 출시 전에 빠진 것 없는지 부서별로 나눠서 봐줘.
```

## 부서 직접 호출

특정 관점만 강하게 보고 싶을 때는 부서명을 직접 부른다.

| 호출어 | 언제 쓰나 |
|---|---|
| `제품관리자` | 기능 범위, 우선순위, 출시 기준을 정할 때 |
| `ui관리자` | 화면 배치, 디자인 시스템, 실제 웹 UI 깨짐을 볼 때 |
| `개발관리자` | 코드 구조, 타입, 유지보수성, 기술 부채를 볼 때 |
| `qa관리자` | 테스트, 회귀, 엣지케이스, 수동 검증을 볼 때 |
| `보안관리자` | 인증, 권한, 데이터 노출, 저장 리스크를 볼 때 |
| `법무관리자` | 약관, 개인정보, 광고/분석/결제 고지를 볼 때 |
| `운영관리자` | 배포, 환경변수, 로그, 장애 대응을 볼 때 |
| `수익화관리자` | Pro, 가격, 광고 제거, 결제 UX를 볼 때 |
| `마케팅관리자` | SEO, 포지셔닝, 전환 문구, 런칭 메시지를 볼 때 |
| `고객지원관리자` | 오류 메시지, 도움말, 문의 흐름을 볼 때 |
| `데이터관리자` | GA4 이벤트, 전환 지표, 개인정보 최소수집을 볼 때 |
| `문서관리자` | 기능 문서, 운영 문서, 릴리즈 노트를 정리할 때 |

## 협업 방식

- 직접 부른 부서가 있더라도, 업무상 필수인 다른 부서 기준이 있으면 함께 언급된다.
- 각 부서는 자기 업무 영역을 벗어나는 부분을 마음대로 처리하지 않고 협업 대상으로 넘긴다.
- 중요한 작업 단위가 끝나면 커밋 권장 여부와 남은 리스크를 보고한다.
- 퇴역 당시 안내는 오래 남아야 할 결정이나 진행 상태를 관련 문서 또는 `work-memory.md`에 기록하는 방식이었다. 현재 장기 메모리는 AutoCorp Studio 세션 메모리를 기준으로 한다.

## 좋은 지시 예시

```text
총괄관리자, 로그인/회원가입을 실제 출시 가능한 수준으로 점검하고 구현까지 해줘.
```

```text
ui관리자, 모바일에서 계산기 화면이 깨지는지 실제 브라우저 기준으로 확인하고 수정해줘.
```

```text
qa관리자, 이번 결제 플로우 변경에 필요한 테스트 시나리오를 만들고 빠진 실패 케이스를 찾아줘.
```

```text
문서관리자, 오늘 결정된 Pro 가격 정책과 남은 법무 리스크를 문서에 정리해줘.
```
