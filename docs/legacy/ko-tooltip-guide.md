# 한국어 툴팁 작성 가이드

선물 계산기 UI의 필드 툴팁·헤더 사용법 툴팁을 쓸 때 따르는 원칙과 구현 규칙을 정리한 문서입니다.

## 핵심 원칙

> **툴팁은 읽는 것이 아니라 훑는 것이다.**

한국어는 영어처럼 단어 단위로 끊기지 않습니다. **의미 덩어리(chunk)** 단위로 줄을 나누고, 정보 계층을 먼저 보이게 한 뒤, 긴 설명은 [이용 가이드](/guide)로 넘깁니다.

| 원칙 | 설명 |
|------|------|
| 의미 단위 줄바꿈 | 조사·구절마다 `<br>`를 넣지 않는다. 한 문장은 가능하면 1~2줄로 이어 붙인다. |
| 계층 분리 | 개요 → 단축키 → (필요 시) 한 줄 팁. 주의·상세 루틴은 가이드 링크로. |
| 중복 금지 | 같은 주의사항을 탭·섹션·각주에 반복하지 않는다. |
| 동일 키 혼란 방지 | Enter 두 번은 `Enter (1회)` / `Enter (2회)`처럼 **순서**를 드러낸다. |
| 가격 필드 | 「현」 버튼 대신 placeholder(현재가) + 스테퍼 시드. 툴팁에만 안내. |

---

## 콘텐츠 3단계

내용량에 따라 배치를 나눕니다.

### 1. 필드 툴팁 (`?` 버튼)

**목표:** 5~15초 안에 역할과 단축키만 파악.

포함:

- `#` 제목 1줄
- 개요 1~2문장 (자연스러운 문장형)
- `────────` 구분선
- `[단축키]` + `Enter (1회) → …` 형식
- `자세히 보기 ↗` → `/guide` (`tooltipGuideLink`)

제외 (가이드로):

- 매일 아침 루틴 상세
- MTS 스냅샷·변동성 주의 장문
- 스테퍼·placeholder 동작의 긴 설명 (한 줄이면 필드에 포함 가능)

**대상 예:** `fields.scenarioPrice`, `fields.accountEquity`, `fields.orderPrice`

### 2. 헤더 「사용법」 툴팁

**목표:** 초보자 / 기존 사용자 중 **하나만** 보여 준다.

- **탭 UI:** `초보자 가이드` | `기존 사용자` (`HowToUseButton.tsx`)
- 탭마다 압축된 본문 1덩어리 + 주의 또는 추천 1블록
- 하단 `자세한 사용법 보기` 버튼
- 초보자 탭에만 통합 주의 (한 종목 가정 등 각주 포함)

### 3. 이용 가이드 페이지 (`/guide`)

툴팁에 넣기엔 긴 루틴, 전제 조건, MTS/HTS 입력 팁, 체크리스트.

## `TooltipBody` 마크업 규칙

문자열은 [`src/i18n/locales/ko.ts`](../src/i18n/locales/ko.ts)의 `hint` / `howToUse.*Body` 등에 두고, [`TooltipBody`](../src/components/TooltipBody.tsx)가 파싱합니다.

| 문법 | 렌더링 |
|------|--------|
| `# 제목` | 굵은 툴팁 제목 |
| 일반 줄 | 문장 블록 (`word-break: keep-all`) |
| `\n\n` | 문단 구분 |
| `────────` | 가로 구분선 |
| `[단축키]` | 섹션 라벨 + 아이콘 (⌨️) |
| `[추천]` / `[주의]` | ✨ / ⚠️ 스타일 라벨 |
| `Enter (1회) → 미리보기` | 키캡(`<kbd>`) + 설명 열 |
| `Ctrl+Z → 반영 취소` | `Ctrl` `+` `Z` 키캡 분리 |
| `- 항목 텍스트` | 불릿 한 줄 |
| `💡 …` | 팁 줄 (기존 `line` 타입) |

`FieldLabelTooltip`에 `guideHref` / `guideLinkLabel`을 넘기면 본문 아래 `자세히 보기 ↗` 링크가 붙습니다.

## 문장 작성: 나쁜 예 vs 좋은 예

### 줄바꿈

**나쁨** — 조사마다 끊김, 세로로 길어짐:

```
계좌 평가금액과
유지·개시 증거금률을 입력한 뒤,
아래 「주문」에서
계약수·가격을 조정하면
체결 직후 상태를
미리 볼 수 있습니다.
```

**좋음** — 문장 단위:

```
계좌 평가금액과 증거금률을 입력한 후, 「주문」에서 계약수·가격을 조정해 체결 직후 상태를 미리 시뮬레이션하세요.
```

### 단축키 (Enter 두 번)

앱 동작은 **같은 Enter 키를 맥락에 따라 두 번** 누르는 것이지, 서로 다른 단축키가 아닙니다.

**나쁨** — 동일하게 보여 혼란:

```
Enter → 미리보기
Enter → 손익 반영
```

**좋음** — 순서 명시 + 키캡 UI:

```
Enter (1회) → 미리보기
Enter (2회) → 손익 반영
```

### 시나리오 가격 필드 툴팁 (슬림 버전)

```
# 시나리오 가격 시뮬레이션
가격 변화에 따른 계좌 상태를 미리 보거나, 매일 시세만 반영해 평가금을 갱신합니다.

────────

[단축키]

Enter (1회) → 미리보기
Enter (2회) → 손익 반영
Esc → 종료
Ctrl+Z → 반영 취소
```

스테퍼·placeholder 안내는 가이드의 「시나리오 가격」 절에 두고, 필드 툴팁에는 넣지 않거나 한 줄로만 언급합니다.

## 가격 입력 UX (툴팁과 짝)

| 항목 | 방식 |
|------|------|
| 시나리오 가격 | `currentPrice`가 있으면 placeholder에 포맷된 현재가 표시 |
| 스테퍼 | `scrubSeedValue={currentPrice}` — 빈 칸에서 ▲▼ 시 현재가 ± 틱 |
| 주문 가격 | 동일 패턴, 「현」 인라인 버튼 없음 |
| 툴팁 | 틱 사이즈 있을 때 위 동작을 `[추천]` 등으로 짧게 안내 |

미입력 시 주문가는 계산상 현재가와 동일하게 간주됨 — `[참고]` 한 줄로 충분.

## CSS 가이드

[`src/App.css`](../src/App.css) — `.tooltip-body`, `.field-label-tooltip`, `.header-how-tooltip`

| 속성 | 권장값 | 이유 |
|------|--------|------|
| `max-width` | `320px` ~ `340px` | 한국어 20~30자 호흡 |
| `max-height` | `80vh` + `overflow-y: auto` | 헤더 툴팁 잘림 방지 |
| `line-height` | `1.5` ~ `1.6` | 너무 넓으면 붕 떠 보임 |
| `letter-spacing` | `-0.02em` ~ `-0.03em` | 한글 UI 밀도 |
| 단축키 | `.tooltip-body__kbd` | 키캡 형태로 시선 고정 |

헤더 사용법 툴팁은 탭(`.header-how-tooltip__tabs`)으로 높이를 절반 이하로 유지합니다.

## 안티패턴 체크리스트

작성·리뷰 시 확인:

- [ ] 한 문장이 4줄 이상으로 세로로 늘어지지 않았는가?
- [ ] 동일 주의 문단이 두 번 이상 반복되지 않았는가?
- [ ] 툴팁만으로 「작은 설명서」가 되지 않았는가? → 가이드로 분리
- [ ] Enter가 두 역할일 때 `(1회)` / `(2회)` 또는 탭·단계로 구분했는가?
- [ ] 초보/숙련 안내를 한 화면에 세로로 다 넣지 않았는가? → 탭
- [ ] 가격 필드에 「현재가 불러오기」 버튼을 다시 넣지 않았는가?
- [ ] 영문 `en.ts`도 같은 구조·슬림 수준으로 맞췄는가?

## 변경 이력 (요약)

1. **의미 단위 줄바꿈** — 조사마다 끊기던 초기 i18n 정리
2. **섹션·단축키 그리드** — `TooltipBody` + `────────` / `[라벨]` / `→`
3. **필드 툴팁 슬림화** — 개요+단축키, `자세히 보기 ↗`
4. **가격 placeholder·스테퍼** — 「현」 버튼 제거, 툴팁에 안내
5. **헤더 사용법 탭** — 초보자/기존 사용자 분리, 문장 압축, 중복 주의 제거

새 필드 툴팁을 추가할 때는 이 문서와 [`calculator-fields.md`](./calculator-fields.md)를 함께 참고하세요.
