# 제품 핵심 설계

상태: 현재 제품 방향의 기준 문서  
적용 범위: 선물 계산기 MVP, 로그인, 저장, 클라우드 동기화, Pro 전환

## 목적

이 문서는 기능 구현 전에 지켜야 할 제품 판단을 고정한다. 코드나 세부 문서가 이 문서와 충돌하면, 먼저 제품 방향을 다시 확인하고 이 문서를 갱신한다.

## 의사결정 담당

제품 방향이 흔들릴 때는 `총괄관리자`에게 맡긴다.

주요 판단은 다음 관점이 함께 본다.

| 주제 | 담당 관점 |
| --- | --- |
| 사용자 가치, MVP 범위, 무료/Pro 경계 | `제품관리자` |
| 저장 위치, RLS, localStorage 위험, 비밀값 | `보안관리자` |
| 개인정보, 약관, 금융 면책, 광고/결제 고지 | `법무관리자` |
| 화면 흐름, 상태 피드백, 선택지 표현 | `ui관리자` |
| 데이터 보관, 삭제, 동기화 정책 | `데이터관리자` |
| 구현 구조, 타입, 테스트, 유지보수 | `개발관리자` |
| 문서 최신성, 레거시 분리 | `문서관리자` |

## 핵심 원칙

1. 계산 도구의 신뢰성이 성장 기능보다 우선이다.
2. 사용자가 입력한 계좌/포지션 숫자는 민감할 수 있으므로 저장은 항상 명시 선택으로 둔다.
3. 로그인은 클라우드 저장을 가능하게 할 뿐, 클라우드 저장을 강제하지 않는다.
4. 사용자가 보안 때문에 로컬 저장을 선택할 수 있어야 한다.
5. 클라우드 동기화는 편의 기능이지 필수 사용 흐름이 아니다.
6. 저장 위치가 바뀌면 UI, 약관, 개인정보 문구도 함께 바뀌어야 한다.
7. 출시 전 완료는 화면 표시가 아니라 저장/복원/삭제/실패/취소 흐름이 검증된 상태다.

## 사용자 상태별 저장 정책

| 사용자 상태 | 기본 저장 | 선택 가능한 저장 | 설명 |
| --- | --- | --- | --- |
| 비로그인 | 저장 안 함 | 이 기기 저장 | `localStorage`에만 저장한다. 서버로 전송하지 않는다. |
| 로그인 | 저장 안 함 | 이 기기 저장, 클라우드 저장 | 로그인해도 기본은 보수적으로 저장 안 함이다. 저장을 켜면 사용자가 저장 위치를 고른다. |
| Pro | 저장 안 함 | 이 기기 저장, 클라우드 저장, 향후 다중 세트 | Pro는 저장 개수와 편의 기능을 확장할 수 있지만 로컬 선택권을 없애지 않는다. |

## 저장 위치별 정책

### 이 기기 저장

- 저장소: 브라우저 `localStorage`
- 장점: 서버로 전송되지 않는다.
- 위험: 같은 기기 사용자, 브라우저 확장, 악성 프로그램, 브라우저 프로필 공유에 노출될 수 있다.
- 삭제: 이 기기 저장을 끄거나 브라우저 사이트 데이터를 삭제한다.

### 클라우드 저장

- 저장소: Supabase `number_sets`
- 장점: 로그인 후 다른 기기에서 복원할 수 있다.
- 보안 기준: Supabase RLS로 본인 row만 읽기/쓰기/삭제할 수 있어야 한다.
- 위험: 사용자가 입력한 계좌/포지션 숫자가 서버 DB에 저장된다.
- 삭제: 클라우드 저장을 끄면 클라우드 저장값만 삭제한다. 로컬 저장값까지 함께 지우면 안 된다.

## 로그인 사용자 저장 UX

로그인 사용자는 저장을 켤 때 저장 위치를 명확히 볼 수 있어야 한다.

- `이 기기`: 보안이 걱정되는 사용자를 위한 기본 선택지.
- `클라우드`: 여러 기기에서 같은 숫자세트를 쓰려는 사용자를 위한 선택지.
- 로컬 draft를 클라우드로 옮기는 동작은 자동이 아니라 명시 버튼으로만 수행한다.
- 저장 위치 전환은 사용자의 선택이어야 하며, 로그인만으로 로컬 저장 버튼이 사라지면 안 된다.

## 문서 구조

현재 기준 문서는 `docs/` 루트에 최소로 둔다.

- `docs/product-core-design.md`: 제품 핵심 설계
- `docs/launch-schedule.md`: 런칭 일정
- `docs/personas/`: 부서 페르소나 기준

그 외 과거 기능 문서와 참고 문서는 `docs/legacy/` 아래에 둔다. 레거시 문서는 참고용이며 최신 제품 기준과 다를 수 있다.

## 남은 결정

- Pro에서 허용할 클라우드 숫자세트 개수
- 클라우드 저장값의 사용자 직접 삭제 UI 범위
- 결제 이후 광고 제거와 저장 기능 확장 경계
- production 도메인 확정 후 약관/개인정보 공개 URL
