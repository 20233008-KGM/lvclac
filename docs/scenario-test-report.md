## 1. 목적

다양한 국적·종목·입력 방식·엣지 케이스에 대해 **수동 산출 정답값**을 정의하고, 계산기(`calculateEvaluate` / `calculateOrder`) 결과와 대조하여 정확성을 검증한다.

---

## 2. 계산 공식 (정답값 산출 기준)

| 항목 | 공식 |
|------|------|
| 포인트가치 | `contractAmount × multiplier ÷ currentPrice` |
| 약정가치(명목) | `contracts × contractAmount × multiplier` |
| 유지증거금 | 명목 × 유지증거금률 **또는** HTS 직접 입력값 (계약수 비례 스케일) |
| 위탁증거금 | 명목 × 위탁증거금률 **또는** HTS 직접 입력값 |
| 버퍼 | `accountEval − maintenanceMargin` |
| 허용 변동폭 | `buffer ÷ (contracts × pointValue)` |
| 롱 청산가 | `currentPrice − 허용 변동폭` |
| 숏 청산가 | `currentPrice + 허용 변동폭` |
| 버틸 수 있는 % | 롱: `(currentPrice − 청산가) ÷ currentPrice × 100` / 숏: `(청산가 − currentPrice) ÷ currentPrice × 100` |
| 레버리지 | `약정가치 ÷ 위탁증거금` |
| 추가매수 가능 | `floor((accountEval − 위탁증거금) ÷ 1계약당 위탁증거금)` |
